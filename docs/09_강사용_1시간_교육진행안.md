# 09. 강사용 1시간 교육 진행안

## 1. 교육 목적

Git을 처음 사용하는 구성원이 1시간 안에 아래 흐름을 직접 수행하도록 합니다.

```text
설치 확인 → 저장소 clone → 파일 수정 → commit → push → GitHub 확인
```

---

## 2. 교육 대상

- Git 사용 경험이 적은 데이터 분석가
- SQL, Python, 보고서 산출물을 GitHub로 관리해야 하는 구성원
- VS Code 기반으로 Git 작업을 수행해야 하는 구성원

## 5. 강사용 설명 스크립트

### 도입

```text
오늘 교육의 목표는 Git 명령어를 모두 외우는 것이 아닙니다.
실무에서 가장 많이 쓰는 수정, commit, push 흐름을 직접 해보는 것입니다.
```

### Git을 쓰는 이유

```text
Git은 파일 백업 도구가 아니라 변경 이력 관리 도구입니다.
누가, 언제, 무엇을, 왜 바꿨는지 확인하기 위해 사용합니다.
```

### commit message 설명

```text
commit message는 나중에 변경 이력을 검색하기 위한 제목입니다.
그래서 '수정', '최종', '작업'처럼 쓰면 안 되고, 변경 성격과 대상을 함께 작성해야 합니다.
```

### 실습 안내

```text
지금부터는 파일 하나를 수정하고 GitHub에 반영해보겠습니다.
흐름은 파일 수정, 상태 확인, add, commit, push 순서입니다.
```

---

## 6. 실습 명령어 모음

```bash
git clone 저장소주소
cd 저장소명
git status
git add 파일명
git commit -m "docs(memo): 업무 메모 수정"
git push origin main
```

---

## 7. 교육 중 강조할 기준

| 기준 | 설명 |
|---|---|
| commit은 작은 단위로 작성 | 여러 목적을 한 commit에 섞지 않음 |
| message는 규칙대로 작성 | `type(scope): 변경 내용` 사용 |
| push 후 GitHub 확인 | 로컬 작업이 원격에 반영됐는지 확인 |
| 업무 자료는 Private 사용 | 고객 데이터, 내부 자료 노출 방지 |

## 8. 교육 후 과제

교육 후 아래 과제를 수행합니다.

```text
1. 개인 레포 생성
2. README.md 수정
3. docs(memo): 자기소개 메모 추가 commit 작성
4. SQL 예제 파일 수정
5. feat(sql): 개인 실습 쿼리 추가 commit 작성
6. GitHub push 후 commit history 확인
```

- 파일 수정
