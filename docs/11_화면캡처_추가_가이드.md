# 11. 화면 캡처 추가 가이드

## 1. 목적

Git 초급자는 텍스트 설명만으로 GitHub와 VS Code 화면을 따라가기 어렵습니다.

교육 자료에 화면 캡처를 추가하면 설치, clone, commit, push, Pull Request 과정을 더 쉽게 따라 할 수 있습니다.

## 5. 캡처 작성 기준

| 기준 | 설명 |
|---|---|
| 개인정보 제거 | 계정명, 이메일, 토큰, 고객명 노출 금지 |
| 화면 확대 | 교육생이 볼 수 있도록 글자 크기 확대 |
| 핵심 영역 표시 | 필요한 경우 빨간 박스 또는 화살표 추가 |
| 파일명 일관성 | 숫자_내용 형식 사용 |
| 한 화면 한 목적 | 여러 기능을 한 캡처에 넣지 않음 |

## 7. 캡처 추가 작업 순서

```text
1. assets/images 폴더 생성
2. 문서별 하위 폴더 생성
3. 화면 캡처 저장
4. 개인정보 제거
5. Markdown 이미지 링크 삽입
6. GitHub에서 이미지 표시 확인
7. commit 작성
```

commit 예시:

```text
docs(capture): Git Bash 설치 화면 캡처 추가
docs(capture): VS Code Source Control 화면 캡처 추가
docs(capture): Pull Request 생성 화면 캡처 추가
```
