# 07. 실무 시나리오 실습

## 1. 목적

Git 명령어를 단순히 연습하는 것이 아니라, 실제 업무에서 자주 발생하는 산출물 변경 상황을 기준으로 실습합니다.

---

## 2. 실습 시나리오 목록

| 시나리오 | 파일 | commit message 예시 |
|---|---|---|
| 업무 메모 작성 | practice/memo/work_note.md | docs(memo): 업무 메모 추가 |
| SQL 쿼리 추가 | practice/sql/sample_query.sql | feat(sql): 사용자 접속 통계 쿼리 추가 |
| Python 분석 코드 추가 | practice/python/sample_analysis.py | feat(python): 접속 로그 집계 코드 추가 |
| 분석 결과 보고서 수정 | practice/report/analysis_result.md | docs(report): 분석 결과 요약 추가 |
| 데이터 검증 로직 추가 | practice/python/sample_validation.py | test(data): 접속 로그 필수 컬럼 검증 추가 |

---

## 3. 시나리오 1. 업무 메모 작성

### 작업 파일

```text
practice/memo/work_note.md
```

### 작업 내용

회의 내용 또는 오늘 작업한 내용을 Markdown으로 작성합니다.

### commit 예시

```bash
git add practice/memo/work_note.md
git commit -m "docs(memo): Git 실습 업무 메모 추가"
```

---

## 4. 시나리오 2. SQL 쿼리 추가

### 작업 파일

```text
practice/sql/sample_query.sql
```

### 작업 내용

사용자 접속 통계 조회용 SQL을 작성합니다.

### commit 예시

```bash
git add practice/sql/sample_query.sql
git commit -m "feat(sql): 사용자 접속 통계 쿼리 추가"
```

---

## 5. 시나리오 3. Python 분석 코드 추가

### 작업 파일

```text
practice/python/sample_analysis.py
```

### 작업 내용

CSV 파일을 읽고 접속 건수를 집계하는 예제 코드를 작성합니다.

### commit 예시

```bash
git add practice/python/sample_analysis.py
git commit -m "feat(python): 접속 로그 집계 코드 추가"
```

---

## 6. 시나리오 4. 분석 결과 보고서 수정

### 작업 파일

```text
practice/report/analysis_result.md
```

### 작업 내용

분석 목적, 기준, 결과 요약을 작성합니다.

### commit 예시

```bash
git add practice/report/analysis_result.md
git commit -m "docs(report): 접속 로그 분석 결과 요약 추가"
```

---

## 7. 시나리오 5. 데이터 검증 코드 추가

### 작업 파일

```text
practice/python/sample_validation.py
```

### 작업 내용

필수 컬럼 존재 여부와 결측치 여부를 검증합니다.

### commit 예시

```bash
git add practice/python/sample_validation.py
git commit -m "test(data): 접속 로그 필수 컬럼 검증 추가"
```

---

## 8. 실무 판단 기준

| 상황 | Type | 이유 |
|---|---|---|
| 새 분석 파일 추가 | feat | 없던 산출물이 추가됨 |
| 잘못된 SQL 조건 수정 | fix | 결과 오류를 바로잡음 |
| README 설명 보완 | docs | 문서만 변경됨 |
| 폴더 구조 정리 | chore | 관리성 작업 |
| 코드 구조 개선 | refactor | 결과 변화 없이 구조만 개선 |
| 검증 코드 추가 | test | 품질 확인 로직 추가 |

---

## 9. 완료 기준

아래 commit을 각각 1회 이상 작성해 봅니다.

```text
docs(memo): Git 실습 업무 메모 추가
feat(sql): 사용자 접속 통계 쿼리 추가
feat(python): 접속 로그 집계 코드 추가
docs(report): 접속 로그 분석 결과 요약 추가
test(data): 접속 로그 필수 컬럼 검증 추가
```
