# 16. GitHub 초대메일 가입 흐름

## 1. 목적

관리자가 교육생 또는 팀원을 GitHub 저장소에 초대했을 때, 교육생이 이메일을 확인하고 초대를 수락하는 과정을 안내합니다.

---

## 2. 전체 흐름

```text
관리자 초대 발송
→ 교육생 이메일 확인
→ GitHub 초대 메일 열기
→ 초대 수락
→ GitHub 로그인 또는 가입
→ 저장소 접근 확인
```

---

## 3. 관리자 작업

관리자는 GitHub 저장소에서 교육생을 collaborator로 초대합니다.

### 절차

1. GitHub 저장소로 이동합니다.
2. `Settings` 메뉴를 선택합니다.
3. `Collaborators` 또는 `Collaborators and teams` 메뉴를 선택합니다.
4. 교육생의 GitHub 계정 또는 이메일 주소를 입력합니다.
5. 초대 권한을 선택합니다.
6. 초대 메일을 발송합니다.

---

## 4. 권한 선택 기준

| 권한 | 사용 기준 |
|---|---|
| Read | 보기 전용 |
| Triage | issue, PR 정리 중심 |
| Write | 실습용 commit, push 가능 |
| Maintain | 저장소 운영 보조 |
| Admin | 저장소 전체 관리 |

교육 실습에서는 일반적으로 `Write` 권한이면 충분합니다.

---

## 5. 교육생 작업

교육생은 초대 메일을 받은 뒤 아래 순서로 진행합니다.

1. 이메일 수신함을 확인합니다.
2. GitHub 초대 메일을 엽니다.
3. 초대 수락 버튼을 선택합니다.
4. GitHub에 로그인합니다.
5. 계정이 없다면 가입을 진행합니다.
6. 초대된 저장소에 접근되는지 확인합니다.

---

## 6. 초대 메일이 보이지 않는 경우

아래 항목을 확인합니다.

| 확인 항목 | 설명 |
|---|---|
| 스팸함 | 초대 메일이 스팸으로 분류될 수 있음 |
| 이메일 주소 | 관리자가 입력한 이메일이 맞는지 확인 |
| GitHub 계정 | 해당 이메일이 GitHub 계정에 등록되어 있는지 확인 |
| 재발송 | 관리자가 초대를 취소 후 다시 발송 |

---

## 7. 저장소 접근 확인

초대 수락 후 아래 항목을 확인합니다.

- 저장소 화면 접근 가능
- Code 버튼 확인 가능
- clone 주소 확인 가능
- 권한에 따라 push 가능

---

## 8. 교육생 안내 문구 예시

```text
GitHub 저장소 초대 메일을 발송했습니다.
메일함에서 GitHub 초대 메일을 확인한 뒤 초대를 수락해 주세요.
GitHub 계정이 없다면 가입과 이메일 인증을 먼저 완료해 주세요.
초대 수락 후 저장소 화면에 접근되는지 확인해 주세요.
```
